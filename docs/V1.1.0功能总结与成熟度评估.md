# V1.1.0 功能总结与成熟度评估

## 功能总结

析灵 SBOM 生成工具 V1.1.0 是一个 Docker 化运行的 Python CLI 工具，入口为 `linx-xiling.py`。工具支持四类扫描目标，并可输出 Linx、SPDX 2.3 和 GB/T 47020-2026 三种 SBOM 格式。

- ISO 镜像扫描：通过 `pycdlib` 直接读取 ISO 文件系统，识别镜像内的 DEB 或 RPM 包，提取包元数据、文件、许可证和依赖关系，不再依赖 FUSE 或容器特权挂载。
- Docker 镜像扫描：支持 Docker Hub 公共镜像和离线 Docker/OCI archive，解析镜像层、处理 overlay whiteout，识别最终 rootfs 中的 dpkg 或 RPM 数据库并生成系统包级 SBOM。
- 单个软件包扫描：支持 `.deb`、`.rpm`、`.src.rpm`、Debian `.dsc`、源码 tar/zip 归档；源码包支持安全解压、文件级许可证扫描和 OSV Scanner 依赖识别。
- 软件源扫描：支持 YUM/DNF 与 APT 仓库元数据发现、下载和解析，生成可从 repo 元数据获得的包级 SBOM。
- 输出格式：支持 Linx 分目录 JSON、SPDX 2.3 JSON、GB/T 47020-2026 `SBOMDF` JSON，并为 GBT 输出生成 `signature.sig` 和 `certification.pem`。
- 配置能力：运行时配置从 `config/config.json` 覆盖默认配置，支持扫描并发、进度条、Docker 平台、源码扫描包含/排除模式、OSV/Elasticsearch 漏洞查询配置。
- 辅助能力：包含供应商、来源方、许可证规则、创建工具元数据、包间关系和文件关系转换逻辑。

## 成熟度评估

按“受控内生产”标准判断，V1.1.0 已具备试生产和内部生产验证基础，但不建议在完全无约束的外部客户环境中直接无条件投产。

可用性较强的理由：

- 主要扫描入口已经具备对应实现和单元测试覆盖，当前 Docker 单测基线为 72 个用例通过。
- ISO 扫描已经从 FUSE/mount 迁移为直接读取，降低了容器特权需求和部署复杂度。
- Docker 镜像扫描同时覆盖在线公共镜像与离线 tar，支持常见 dpkg/RPM 数据库路径。
- 输出层已覆盖 Linx、SPDX 2.3、GBT 三种交付格式，并有关键字段映射测试。
- 配置解析具有逐项校验和回退能力，错误配置不会轻易中断扫描。

仍需收口的风险：

- CLI 原先在未提供 `--format` 时默认输出 Linx+SPDX，容易造成用户误用；V1.1.0 收尾阶段改为 `--format` 必填。
- `docker-compose.yml` 仍残留旧 FUSE/特权配置和 `1.0.0` 镜像名，需要与直接 ISO 扫描和 V1.1.0 交付保持一致。
- 测试集中在单一大文件中，可读性和后续扩展性不足，需要按功能域拆分。
- GBT 漏洞查询依赖本地 Elasticsearch/OSV 数据库，离开受控环境时需要明确部署前置条件。
- 真实集成测试依赖大 ISO、网络 repo、Docker Hub 和外部样本，执行时间和环境波动高，需要分层运行和清理策略。

## 结论

V1.1.0 可以进入受控内生产验证：适合在已准备 Docker 镜像、配置文件、网络访问和 OSV/Elasticsearch 服务的环境中使用。正式对外发布前，建议完成本次收尾计划中的 CLI 行为修正、部署配置清理、测试拆分、真实样本集成验证和版本化文档更新。
